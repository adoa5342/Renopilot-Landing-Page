# RenoPilot.Website.LandingPage

## High Level System

The system follows a typical Payload CMS project structure, with SQLite currently being used as a lightweight database, storing all CMS content, including current pages and their layouts, block data, user information, and uploaded media. During development the project uses a local SQLite file (renopilot.db), but in production this will be swapped for a scalable database such as Postgres or MongoDB.

Content is added by admins via the Payload CMS UI and stored in the database. At runtime, when the webpage is opened, the Next.js frontend fetches the content from Payload and renders it for the end-user.

## Codebase

### Payload config: `src/payload.config.ts`
The payload config defines the database adapter (SQLite via DATABASE_URI), collections (Users, Media, Pages), and available blocks, and instructs Payload on how to store and structure content.

### Blocks: `src/blocks/*`
Each block schema specifies what fields are available in the CMS i.e. what content editors can fill in for that block inside the CMS.

For example, HeroBlock contains a heading, subheading, and background image; ContentBlock contains text, image, and a CTA link; InteractiveBlock holds before/after images; and ImageGridBlock stores a set of labeled images. This separation makes the system modular and easy to extend.

### `src/app/*` (Routes — App Router)
Public-facing routes (each folder with a `page.tsx` becomes a route).

A shared **Page Rendering** component iterates over the `layout` array and renders the correct React block for each `blockType`.

### Frontend rendering: `src/app/(frontend)`
 React components that visually render each CMS block. They are mpped by `blockType` in the page document’s `layout` array.

For example, the homepage (page.tsx) fetches the pages collection with the slug "landing-page". It iterates through the layout array and maps each block type to its corresponding React component in /components (e.g., HeroBlock.tsx, ContentBlock.tsx). These components handle the actual visuals and are wrapped by layout.tsx, which provides global HTML structure and styling.

### Admin app: `src/app/(payload)`
Auto-generated by Payload, this provides the admin dashboard at /admin, where editors can log in, create users, upload media, and build pages.

# Local Development

1. **Install dependencies**
   ```bash
   pnpm install
   ```
2. **Set environment**
   ```bash
   cp .env.example .env  # if available
   # ensure DATABASE_URI and PAYLOAD_SECRET are set
   ```
3. **Run the app**
   ```bash
   pnpm dev
   ```
4. **Open Admin**
   - Visit **http://localhost:3000/admin** to log in and manage content.
